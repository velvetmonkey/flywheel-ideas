## Initial stance (Pass 1)

The failure of Microsoft Bob was a direct consequence of its core premise being flawed. The assumption that a residential-house metaphor would 'map cleanly' onto computer functions for novice users was an oversimplification that actively worked against user comprehension. Instead of a faster, more intuitive experience, users were forced to translate abstract computing concepts into a literal, often awkward, domestic environment. Trying to find a file wasn't like putting something in a drawer; it was a convoluted process of navigating 'rooms' to find the 'desk' with the right 'drawer,' all while hoping the metaphor didn't break down for more complex operations. This added cognitive load didn't accelerate learning; it introduced a new, unnecessary hurdle, making Bob slower and more confusing than the abstract interfaces it aimed to replace. The system tried too hard to be 'friendly' by resorting to a literal metaphor, which in turn made it brittle and inefficient for actual task completion.
